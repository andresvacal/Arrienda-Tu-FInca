Casos de Prueba para el Backend

API de Registro de Usuario

Objetivo: Verificar que la API registra correctamente a un nuevo usuario.

Request: POST a /arrendadores/registro con los datos del usuario.

Resultado esperado: Se crea un nuevo usuario en la base de datos, y se devuelve un código de estado 201 con la información del usuario registrado.

API de Autenticación

Objetivo: Comprobar que la API autentica correctamente a un usuario existente.

Request: POST a /auth/login con credenciales válidas.

Resultado esperado: Se devuelve un código de estado 200 con un token de autenticación.

API de Listado de Propiedades

Objetivo: Confirmar que la API devuelve un listado de propiedades disponibles.

Request: GET a /propiedades/buscar.

Resultado esperado: Se devuelve un código de estado 200 con un array de propiedades disponibles.

API de Creación de Solicitud de Arriendo

Objetivo: Verificar que la API procesa correctamente las solicitudes de arriendo.

Request: POST a /solicitudes/nueva con los detalles de la solicitud.

Resultado esperado: Se crea una nueva solicitud en la base de datos, y se devuelve un código de estado 201 con los detalles de la solicitud creada.
